# cfneic

`cfneic` updates MERMAID preliminary event metadata with reviewed ISC-EHB/NEIC hypocenters while preserving the legacy Fortran processing behavior.

The supported workflow is the `run_cfneic` wrapper. It reads a flat input catalog, builds local GPS products with `rdGPS`, runs `cfneic`, and writes all generated products under an `outputs/` directory.

Originally written by Guust Nolet. 

Further developed by [Joel D. Simon](https://github.com/jdsimon) for [Bathymetrix](https://github.com/bathymetrix) (<jdsimon@bathymetrix.com>).

## Supported workflow

Use `run_cfneic` with a flat input directory containing:

```text
*geo*.csv
neic.csv
ehb.hdf
tomocat.txt
```

Example layout:

```text
~/mermaid/cfneic/
├── inputs/
│   ├── 452.020-P-06_geo_DET.csv
│   ├── 452.020-P-07_geo_DET.csv
│   ├── neic.csv
│   ├── ehb.hdf
│   └── tomocat.txt
└── outputs/              # generated by run_cfneic
```

`run_cfneic` discovers floats from the `*geo*.csv` files present in `--input-root`. It does not require the old `452*/` processed-directory layout for normal runs.

## Float discovery

Float IDs are resolved from each GeoCSV file in this order:

1. Preferred: the GeoCSV `Station` column.
   - Example: `Station=P0006` becomes `GPS.06`.
   - Leading zeroes are reduced to the legacy two-character GPS ID where applicable.

2. Fallback: filename patterns such as `-P-06` or `-N-03`.
   - This is only used when no usable `Station` value is available.
   - Dry-run output marks this as `filename fallback`.

The `Station` column is therefore the current authoritative source for GPS IDs; filename parsing is compatibility behavior.

## Prerequisites

- `zsh`
- `make`
- `gfortran`
- A flat input catalog containing `*geo*.csv`, `neic.csv`, `ehb.hdf`, and `tomocat.txt`

The Fortran source is under `src/fortran/`. Helper scripts are under `scripts/`. Test and regression fixtures are under `tests/`.

`rdGPS` supports both the legacy GeoCSV layout and expanded GeoCSV layouts with additional columns such as `DataQuality` and `SampleCount`, as long as the required columns are present by name.

## Build

From the repository root:

```sh
make
```

This builds:

```text
build/rdGPS
build/cfneic
```

The source order and compiler commands are managed by the `Makefile`; do not compile the Fortran files individually for normal use.

## Preflight / dry run

Inspect paths, GeoCSV discovery, NEIC first-row handling, and GPS ID resolution without generating outputs:

```sh
./run_cfneic --dry-run
```

With explicit paths:

```sh
./run_cfneic \
  --input-root /path/to/inputs \
  --output-dir /path/to/cfneic-run-root \
  --dry-run
```

Typical dry-run output includes lines like:

```text
GPS.06 <= 452.020-P-06_geo_DET.csv (Station=P0006)
```

or, when the Station column is unavailable:

```text
GPS.06 <= 452.020-P-06_geo_DET.csv (filename fallback)
```

Dry-run also reports whether `neic.csv` appears to have a header or begins with a date row.

## Full run

Default paths are:

```text
--output-dir  ~/mermaid/cfneic
--input-root  ~/mermaid/cfneic/inputs
```

Run with defaults:

```sh
./run_cfneic
```

Run with explicit paths:

```sh
./run_cfneic \
  --input-root /path/to/inputs \
  --output-dir /path/to/cfneic-run-root
```

`run_cfneic` will:

1. verify required input paths exist;
2. remove and recreate `--output-dir/outputs`;
3. build local executables with `make` if needed;
4. write `input_manifest.before`;
5. copy `neic.csv`, `ehb.hdf`, and `tomocat.txt` into the run directory;
6. convert the first five NEIC CSV fields into `neic.txt`;
7. run `rdGPS` once for each discovered `*geo*.csv` file;
8. move header-only GPS files into `empty_gps/`;
9. write `dumgps` from the remaining non-empty `GPS.*` files;
10. run `cfneic tomocat.txt` from the output directory;
11. write `input_manifest.after` and fail if input checksums changed during the run.

Input files under `--input-root` are not modified. Treat `outputs/` as disposable generated output; every non-dry run recreates it.

## Command-line options

```text
Usage:
  run_cfneic [--output-dir DIR] [--input-root DIR]
             [--tomocat FILE] [--neic FILE] [--ehb FILE]

Options:
  -i, --input-root DIR   Flat input directory containing *geo*.csv, neic.csv,
                         ehb.hdf, and tomocat.txt.
                         Default: output-dir/inputs
  -o, --output-dir DIR   Run root containing inputs/ and outputs/ subdirs.
                         Generated files go directly to output-dir/outputs.
                         Default: ~/mermaid/cfneic
      --tomocat FILE     tomocat input file. Default: input-root/tomocat.txt
      --neic FILE        NEIC CSV file. Default: input-root/neic.csv
      --ehb FILE         ISC-EHB HDF file. Default: input-root/ehb.hdf
      --dry-run          Show resolved paths and input preflight, then exit.
  -h, --help             Show help.
```

## Outputs

Generated files are written under:

```text
--output-dir/outputs/
```

Important outputs include:

```text
out.cfneic_trig      triggered records with updated hypocenters and positions
out.cfneic_int       interpolated records with updated hypocenters and positions
hypos                nearby catalog events that could be missed events
missed_events        records that could not be matched to a usable catalog/GPS pair
log.cfneic           cfneic run diagnostics
errors.log           recoverable wrapper and Fortran data issues
neic.txt             wrapper-generated first-five-column NEIC text file
dumgps               list of usable GPS.* files consumed by cfneic
GPS.*                per-float rdGPS products
path*.xy             per-float trajectory plotting coordinates
out.rdGPS*           rdGPS diagnostics
empty_gps/GPS.*      header-only GPS files excluded before cfneic
input_manifest.*     before/after checksums of the input catalog
```

`out.cfneic_trig` contains events that triggered an immediate surfacing. `out.cfneic_int` contains the other, interpolated events.

Selected output columns:

```text
year jd hr mi s ms   reviewed ISC/NEIC origin time
evlo evla evdp       reviewed ISC/NEIC hypocenter
kstnm                station code
stlo stla            station longitude and latitude
gcarc                great-circle distance from reviewed hypocenter
tobs                 observed travel time adjusted for updated event time
stder                error estimate from tomocat.txt
tasc                 time between trigger and surfacing
snr                  signal-to-noise ratio from tomocat.txt
ocdp                 ocean depth from tomocat.txt
stel                 station elevation, negative MERMAID depth from tomocat.txt
p                    P-wave slowness; upgoing p has negative slowness
v1 v2                average velocities over previous and current legs
acc                  average acceleration over the two legs
b h locnerr          location-error terms from the MERMAID location workflow
```

See `docs/tobs.info` for more about `tobs`.

## Verification targets

### GeoCSV discovery

Verifies that `run_cfneic` discovers floats from flat `*geo*.csv` files in `--input-root`, independent of legacy serial-prefix directories, and that the GeoCSV `Station` column is preferred over filename fallback.

```sh
make verify-geocsv-discovery
```

Optional output override:

```sh
VERIFY_GEOCSV_DISCOVERY_OUT=/tmp/cfneic-geocsv make verify-geocsv-discovery
```

### rdGPS fixtures

Verifies that `rdGPS` parses both legacy and expanded GeoCSV fixture layouts.

```sh
make verify-rdgps-fixtures
```

Optional output override:

```sh
VERIFY_RDGPS_OUT=/tmp/cfneic-rdgps make verify-rdgps-fixtures
```

### NEIC legacy behavior

Verifies the intentionally preserved first-row behavior for headered and headerless `neic.csv` files.

```sh
make verify-neic-legacy-behavior
```

Optional output override:

```sh
VERIFY_NEIC_OUT=/tmp/cfneic-neic make verify-neic-legacy-behavior
```

### Legacy output regression

Regenerates outputs and compares them against frozen reference products in `tests/fixtures/legacy_outputs/`.

```sh
make verify-legacy-outputs
```

By default this uses:

```text
VERIFY_BASELINE=tests/fixtures/legacy_outputs
VERIFY_INPUT=/Users/jdsimon/mermaid/cfneic/inputs
```

Override these when needed:

```sh
VERIFY_INPUT=/path/to/inputs \
VERIFY_OUT=/tmp/cfneic-verify \
make verify-legacy-outputs
```

The legacy-output fixtures are regression baselines only; they are not active run products.

## Known legacy NEIC first-row behavior

`run_cfneic` converts `neic.csv` to `neic.txt` with:

```text
first five CSV fields only
```

The legacy `cfneic` program then skips the first line of `neic.txt`.

Consequences:

- If `neic.csv` has a header, the header is skipped.
- If `neic.csv` has no header, the first data row is skipped.

This is historical behavior and is intentionally preserved by the current tests. Use:

```sh
./run_cfneic --dry-run
```

to see which case the wrapper detects, and:

```sh
make verify-neic-legacy-behavior
```

to protect the current behavior before changing it.

## Troubleshooting

### `filename fallback`

Dry-run example:

```text
GPS.06 <= 452.020-P-06_geo_DET.csv (filename fallback)
```

Meaning: the wrapper could not find a usable `Station` value in the GeoCSV data and derived the GPS ID from the filename.

Preferred fix: populate the GeoCSV `Station` column. Filename fallback is retained for compatibility, not as the primary workflow.

### Duplicate station IDs

Preflight example:

```text
ERROR: Preflight: duplicate station id '06' from /path/to/file.csv
```

Meaning: two or more GeoCSV files resolved to the same GPS ID. Check both the `Station` column and filename fallback patterns. Each discovered float must resolve to a unique ID.

### Missing `*geo*.csv`

Preflight example:

```text
ERROR: No *geo*.csv files found under /path/to/inputs
```

Meaning: `--input-root` exists, but contains no files matching `*geo*.csv`.

Check that the input path is correct and that GeoCSV filenames contain the substring `geo` and end in `.csv`.

### No station ID

Preflight example:

```text
ERROR: Preflight: no station id for /path/to/file.csv
```

Meaning: no usable `Station` value was found and the filename did not contain a fallback pattern like `-P-06` or `-N-03`.

Fix the GeoCSV `Station` column or rename the file to include a legacy-compatible identifier.

### Empty GPS files

Legacy `cfneic` stops if a `GPS.*` file contains no usable data. The wrapper avoids this by moving header-only GPS files into:

```text
outputs/empty_gps/
```

and logging the condition in:

```text
outputs/errors.log
```

If all GPS files are empty, the run fails with:

```text
ERROR: No non-empty GPS files remain after rdGPS processing.
```

### Input manifest changes

The wrapper writes checksums before and after the run:

```text
input_manifest.before
input_manifest.after
```

If they differ, the run fails with:

```text
ERROR: Input files changed during run
```

This protects reproducibility. Re-run after confirming that no external process is modifying files under `--input-root` while `run_cfneic` is active.

### Missing required inputs

Setup errors such as missing `neic.csv`, `ehb.hdf`, `tomocat.txt`, or `--input-root` are fatal. Confirm the flat input directory contains all required files, or pass explicit paths with `--tomocat`, `--neic`, and `--ehb`.

## Legacy notes

The historical workflow operated from a run root containing MERMAID processed directories such as `452*/`, edited or compiled source paths manually, ran `scripts/rdGPS_all`, and then invoked `cfneic` directly.

That old directory-oriented workflow is retained only as legacy context. The current supported workflow is:

```sh
./run_cfneic --input-root /path/to/flat/inputs --output-dir /path/to/run-root
```

where `--input-root` contains flat `*geo*.csv`, `neic.csv`, `ehb.hdf`, and `tomocat.txt` files.

`rdGPS` still has a legacy directory mode:

```text
rdGPS FLOAT_DIR NR [root]
```

but normal repository use should go through `run_cfneic`.

Additional historical notes are preserved in `docs/README_legacy`.
